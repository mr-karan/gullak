# Gullak

Gullak is a local-first mobile expense tracker for fast personal logging,
bank-SMS review, and optional self-hosted sync. The phone owns the ledger in an
on-device SQLite database; the server is a merge point and trusted AI box, not a
required runtime dependency.

## What works in 0.1.0

- Flutter Android app with Drift/SQLite as the source of truth.
- Onboarding that seeds accounts, category groups, and emoji-backed categories.
- Manual Quick Entry for expenses, income, transfers, split transactions, and
  receipt/image-assisted entry.
- Activity, Budget, Inbox, Accounts, Recurrences, Categories, Settings, backup,
  import/export, and sync screens.
- Local change log + push/pull sync to a Bun/Hono/SQLite server.
- Server-side AI parsing for Quick Entry, SMS, WhatsApp, and feedback debugging.
- SMS Inbox for classifier-positive bank messages with confirm/dismiss flows,
  category/account hints, retryable rescans, parser-debug dialogs, and “Send
  feedback” uploads to the sync server.
- WhatsApp bridge based on Baileys, backed by a single SQLite auth/cache store.

## Architecture

```
┌────────────────────┐      HTTPS       ┌────────────────────────┐
│ Flutter app         │◀───────────────▶│ pi-server              │
│ Drift + SQLite      │ sync / AI JSON  │ Bun + Hono + Drizzle   │
│ local source truth  │                 │ bun:sqlite             │
└────────────────────┘                 └───────────▲────────────┘
                                                    │ webhook
                                      ┌─────────────┴─────────────┐
                                      │ whatsapp-bridge            │
                                      │ Bun + Baileys + SQLite     │
                                      └───────────────────────────┘
```

| Component | Path | Purpose |
| --- | --- | --- |
| Mobile app | `app/` | Flutter, Riverpod, Drift, SMS reader, Quick Entry UI |
| Sync/API server | `pi-server/` | Hono API, Drizzle schema, sync merge point, AI endpoints |
| WhatsApp bridge | `whatsapp-bridge/` | Baileys socket, inbound webhook, outbound send API |

## Data model rules

- **Money** is always integer minor units (`₹490.00` → `49000`).
- **IDs** are UUID strings generated by clients and accepted by the server.
- **Dates** are `YYYY-MM-DD`; timestamps are Unix epoch milliseconds.
- **Local-first** means Drift writes locally first, then sync pushes/pulls rows.
- **Conflict policy** is last-write-wins by `updatedAt`, appropriate for a small
  personal/family deployment.

## Quick start

### Mobile app

```bash
cd app
flutter pub get
flutter run
```

From repo root, useful shortcuts:

```bash
just devices      # list connected Android devices
just apk          # build release APK into app/dist/
just install      # build and adb install latest APK
just clear-data   # wipe dev.mrkaran.gullak app state on the connected phone
just launch       # launch installed app
just gate         # pre-commit app checks
```

### pi-server

```bash
cd pi-server
bun install
bun run db:generate   # only when schema changes
bun run dev           # hot reload on :8787
bun run typecheck
bun run start
```

The server auto-applies Drizzle migrations at startup. Default DB path is
`../data/gullak.db`; set `GULLAK_DB_PATH` to override.

### WhatsApp bridge

```bash
cd whatsapp-bridge
bun install
bun run start
```

Set `WEBHOOK_URL` to the pi-server webhook. The production homelab deployment
mounts `/mnt/storage/gullak` and stores bridge auth/cache in `whatsapp.db`.

## Configuration

### pi-server environment

| Variable | Purpose |
| --- | --- |
| `GULLAK_DB_PATH` | SQLite file path, default `../data/gullak.db` |
| `GULLAK_HOST`, `GULLAK_PORT` | HTTP bind address, default `127.0.0.1:8787` |
| `GULLAK_HTTP_API_KEY` | Enables `x-api-key` auth for non-public endpoints |
| `GULLAK_MODEL_BASE_URL` | OpenAI-compatible chat completions base URL |
| `GULLAK_MODEL_ID` | Model id used by AI extraction/agent endpoints |
| `GULLAK_MODEL_API_KEY` | Model provider API key |
| `OPENROUTER_API_KEY` | Accepted alias; defaults base/model for OpenRouter |
| `OPENAI_API_KEY` | Accepted alias; defaults base/model for OpenAI |
| `GULLAK_WHATSAPP_BRIDGE_URL` | Internal bridge URL for outbound WhatsApp replies |

If no hosted model key is present, the server falls back to a local Ollama-style
OpenAI-compatible endpoint.

### App configuration

Configure sync in `Settings → Sync server`:

- Base URL, e.g. `https://gullak.mrkaran.dev`
- API key matching `GULLAK_HTTP_API_KEY`

The app never stores OpenRouter/OpenAI credentials. AI-backed Quick Entry, SMS
parse, receipt parse, feedback upload, and WhatsApp agent work through the sync
server.

when `GULLAK_HTTP_API_KEY` is set.

## Sync model

1. App repositories write normal tables and append local `change_log` rows via
   `ChangeLogWriter`.
2. `SyncService.pushPending` batches unsynced mutations to `/v1/sync/push`.
3. Server applies the row changes and appends server `change_log` rows inside
   one transaction.
4. `SyncService.pullChanges` pages `/v1/sync/changes` and applies remote rows
   directly to Drift via `RemoteApplier`.
5. The server filters changes from the same `clientId` to avoid echoing a
   client's own writes.

## SMS and AI parsing

- Android SMS ingestion uses a local classifier first, then asks pi-server to
  parse classifier-positive messages.
- `POST /v1/ai/sms/parse` returns `SmsCandidate` JSON: amount, direction,
  payee, account hint, category hint/id, date, bank reference, and confidence.
- The app sends local categories and learned payee→category mappings with each
  parse request. Server resolution order is:
  1. learned payee mapping,
  2. LLM-returned category,
  3. deterministic merchant fallback rules,
  4. `null` / uncategorised.
- Inbox refresh deletes retryable local stale rows (`error`, `none`,
  `duplicate`) and clears old parse cache before rescanning. This avoids the
  Android SMS id dedupe trap where old parser failures were never reparsed.
- Failed Inbox cards expose a robot debug dialog and can upload structured
  feedback to `/v1/feedback` for developer inspection.

## UI conventions

- Category icons are emoji-first. Activity rows show the category emoji when a
  category is present, transfer/split icons otherwise.
- Snackbars must use `showTimedSnackBar` from `app/lib/core/snackbars.dart`.
  This force-closes snackbars after a short duration so Android accessibility
  timing quirks cannot leave stale toasts stuck onscreen.

## Resetting for a fresh start

Remote homelab DB reset should keep a backup first:

```bash
ssh floyd-homelab-1 'cd /mnt/storage/gullak && cp gullak.db gullak.db.backup-$(date +%Y%m%d-%H%M%S) && rm -f gullak.db gullak.db-wal gullak.db-shm'
```

Then restart the stack so migrations recreate an empty DB.

Android app state reset:

```bash
just clear-data
```
